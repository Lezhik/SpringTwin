# Spring Twin CLI

Spring Twin CLI --- инструмент анализа архитектуры Java / Spring Boot
проектов.

Он строит граф зависимостей между классами на основе: - Spring
Dependency Injection (анализ исходников) - анализа байткода (.class)

Далее граф используется для кластеризации проекта и генерации задач для
AI‑агентов (Cursor AI, Kilo Code) для архитектурного рефакторинга.

------------------------------------------------------------------------

## Основные возможности

1.  **Анализ исходного кода**
    -   поиск Spring компонентов (`@Controller`, `@Service`,
        `@Repository`, `@Component`, `@Configuration`, `@Bean`)
    -   извлечение зависимостей DI
    -   определение constructor injection
2.  **Анализ байткода**
    -   field references
    -   method calls
    -   object instantiation
    -   связи между классами
3.  **Кластеризация архитектуры**
    -   объединение графов зависимостей
    -   выделение логических модулей проекта
    -   поиск межкластерных связей
4.  **Генерация задач для AI**
    -   задачи для рефакторинга
    -   уменьшение связности между кластерами
    -   разбиение крупных классов

------------------------------------------------------------------------

## Установка

Требования:

-   Java 21+
-   Gradle 8+

Сборка:

    ./gradlew build

Запуск CLI:

    java -jar spring-twin-cli.jar

------------------------------------------------------------------------

## Команды

### 1. Анализ Spring исходников

    spring-twin scan-source   --project <path>   --output <di.json>   --include <packageMask>   --exclude <packageMask>

Результат: JSON граф DI зависимостей.

------------------------------------------------------------------------

### 2. Анализ байткода

    spring-twin scan-bytecode   --classes <path>   --output <bytecode.json>   --include <packageMask>   --exclude <packageMask>

Результат: JSON граф зависимостей из байткода.

------------------------------------------------------------------------

### 3. Кластеризация

    spring-twin cluster   --di <di.json>   --bytecode <bytecode.json>   --output <clusters.json>

Результат: - список кластеров (логических модулей) - метрики качества
кластеризации - связи между кластерами.

------------------------------------------------------------------------

### 4. Генерация задач для AI

    spring-twin generate-refactoring   --clusters <clusters.json>   --output <tasks.json>

Результат: список задач для AI‑агентов по архитектурному рефакторингу.

------------------------------------------------------------------------

## Формат графа (упрощённо)

    nodes:
      - Class

    edges:
      - DEPENDS_ON
      - CALLS
      - INSTANTIATES
      - ACCESSES_FIELD

Каждая связь содержит детали (например тип инъекции, поле или метод).

------------------------------------------------------------------------

## Ограничения MVP

-   анализ только Java
-   внешние библиотеки не индексируются
-   runtime анализ отсутствует

------------------------------------------------------------------------

Spring Twin помогает LLM и разработчикам лучше понимать архитектуру
проекта и планировать безопасный рефакторинг.
