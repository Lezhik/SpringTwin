# Spring Twin

Понимай архитектуру. Управляй сложностью.

---

## О проекте

Spring Twin — MCP-агент для анализа Spring Boot проектов. Проект извлекает архитектурный контекст, строит граф зависимостей и позволяет формировать explain-отчеты через веб-интерфейс и интеграции с Cloud LLM.

Проект не является полноценным цифровым двойником на текущем этапе, но закладывает архитектурный фундамент для дальнейшего развития.

---

## Особенности MVP

- Индексация Spring Boot проекта (анализ Java AST)
- Построение DI-графа зависимостей
- Расширение графа через анализ байткода (.class файлы)
- explain-отчеты по endpoint, классу, методу
- Веб-интерфейс для просмотра графа и генерации отчетов
- Поддержка in-memory Neo4j
- Маска include/exclude пакетов для контроля области анализа
- Использование меток (labels) для Spring аннотаций

---

## Технический стек

**Backend:** Java, Spring Boot, WebFlux, Neo4j (in-memory)  
**Frontend:** TypeScript, Vue.js  
**Сборка:** Gradle/Kotlin DSL  
**Тестирование:** JUnit (backend), Vitest (frontend), Cypress (e2e)

---

## Вектор развития

- Поддержка standalone Neo4j
- Хранение текстовых embeddings и семантический поиск
- Детекция архитектурных аномалий
- Расширенный call graph и runtime-анализ
- Кластеризация проекта и поддержка рефакторинга
- Расширение меток за счет стандартных архитектурных шаблонов (Factory, Event, Listener, Adapter и др.)

## Установка и запуск (MVP)

1. Клонируйте репозиторий:
```bash
git clone <repository_url>
```
2. Перейдите в папку проекта:
```bash
cd spring-twin
```
3. Сборка проекта:
```bash
gradle build
```
4. Запуск backend сервера:
```bash
gradle bootRun
```
5. Запуск frontend:
```bash
gradle frontendRun
```
6. Откройте веб-интерфейс: `http://localhost:8080`

---

## Конфигурация агента

- includePackages: список пакетов для индексации
- excludePackages: список пакетов для исключения из индексации
- режим Neo4j: по умолчанию in-memory (standalone отложен на будущие фазы)

---

## Использование

1. Настройте include/exclude пакеты
2. Запустите индексацию проекта
3. Просмотрите граф зависимостей
4. Используйте explain для endpoint, класса или метода
